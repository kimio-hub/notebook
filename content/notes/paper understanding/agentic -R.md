# Agentic-R 论文理解笔记

## 论文要解决什么问题
这篇工作的核心目标，是让 deep research agent 不只是“会搜索”，而是能和检索模块形成互相促进的闭环。作者关注的问题是：在多轮 reasoning + query + retrieval + answer 的流程中，检索器和 agent 往往彼此制约，如果只单独优化其中一部分，整体效果未必最好。

## 核心思路 / 方法
按照我目前的理解，这篇论文的关键思路是把 **agent** 和 **retrieval** 看成一个联合系统，然后交替优化。

整体流程可以理解为：
1. 输入一个问题；
2. agent 先进行一轮 reasoning；
3. 根据当前思考生成 query；
4. 检索器返回 passages；
5. 经过多轮交互后，agent 生成最终答案。

作者进一步关注的是：如何从这些交互轨迹中构造训练信号，既优化检索器，也优化 agent。

## 训练或实验设置
这项工作大致包含两个交替阶段：

### 1. 检索器训练
在检索器训练阶段，每个 query 会对应多个 candidate passages。为了筛选更好的训练样本，作者设计了一个打分过程：
- 先结合问题、当前思考链和 query，生成一个 **subanswer**；
- 再根据 subanswer 对 candidate passages 做打分；
- 另一部分打分则是：把某个 passage 接回 agent 的后续推理，如果它能帮助模型最终走到正确答案，就给高分，否则给低分。

这些信号会被综合起来，用于构造正负样本，从而训练 retrieval 模块。

### 2. Agent 训练
当 retrieval 模块有一定能力后，再固定检索器，去训练 agent。这一阶段使用了 PPO 风格的强化学习方法，让 agent 学会更好地发起 query、利用检索结果并完成最终回答。

## 易混点 / 修正点
下面几个点需要特别注意：

- **E5 检索器不是从零开始训练出来的，而是预先准备好的基础检索器。**
- **subanswer 不是某个 passage，本质上更接近一个中间参考答案或中间监督信号。**
- **交替训练顺序要注意区分。** 我一开始理解成“先训练 retrieval，再训练 agent”，但修正点里强调的是：更准确的理解应当是 agent 与 retrieval 交替优化，而不是简单单向流水线。

## 我的理解
我觉得这篇论文的重要性在于：它不是把检索看成固定工具，而是把检索器本身也放进 agent 系统里共同训练。这样做的意义是，agent 产生什么样的 query，会反过来影响检索器应当擅长什么；而检索器返回什么样的证据，又会影响 agent 后续的 reasoning 路径。

换句话说，这篇工作强调的是 **agent 与 retrieval 的协同优化**，而不是“先把检索做好，再把 agent 接上去”。

## 小结
这篇论文可以理解为一种面向 deep research 场景的联合优化框架：通过多轮 reasoning 和 retrieval 轨迹构造训练信号，交替提升 agent 与 retrieval 的能力。对我来说，理解这篇论文的关键不是记住所有细节，而是抓住三点：
- 它研究的是 **agent 与检索协同训练**；
- 它依赖 **轨迹级打分与样本构造**；
- 它最终希望提升的是复杂检索推理任务中的整体闭环效果。
