# ReAct 学习笔记

## 摘要
ReAct（Reason + Act）强调让模型在任务过程中交替进行“思考（Thought）”与“行动（Action）”。它不是只在内部推理，也不是只与外部工具交互，而是把两者结合起来，形成“思考—行动—观察—再思考”的闭环。

## 核心思想
传统语言模型往往只根据上下文直接生成答案；而在 ReAct 范式中，模型可以先进行内部思考，再调用外部动作（如搜索、计算器、数据库查询等），接收环境反馈后继续推理。这样做的好处是：模型既能保留推理能力，又能借助外部环境补充信息。

## 关键概念
- **Question / Input $x$**：用户最初提出的问题。
- **Action Space $A$**：模型可执行的外部动作，例如 `Search(query)`、`Calculator(expr)`。
- **Observation Space $O$**：环境对动作返回的结果。
- **Thought Space $T$**：模型的内部推理内容。

在 ReAct 中，可以把“思考”和“动作”看成一个扩展后的决策空间，因此模型每一步都要决定：下一步是继续想，还是采取行动。

## 上下文表示
如果只考虑动作与观察，在第 $k$ 步之前的上下文可以写成：

$$
c_k=(x,a_1,o_1,a_2,o_2,\dots,a_{k-1},o_{k-1})
$$

而在 ReAct 中，由于加入了 thought，上下文会扩展为：

$$
c_k=(x,t_1,a_1,o_1,t_2,a_2,o_2,\dots,t_{k-1},a_{k-1},o_{k-1})
$$

这表示模型不仅记住做过什么，还记住“为什么这么做”。

## 轨迹生成
一条 ReAct 轨迹可以理解为 Thought、Action、Observation 的交替序列。若环境记为 $E$，则联合过程可以近似写成：

$$
P(\tau\mid x)=\prod_{k=1}^{N} P(t_k\mid c_k)\, P(a_k\mid c_k,t_k)\, P(o_k\mid a_k,E)
$$

其中：
- 先根据当前上下文生成 thought；
- 再根据 thought 选择 action；
- 最后由环境返回 observation。

## 与其他范式的关系
- **只做 CoT**：有推理，但不主动与外部环境交互。
- **只做工具调用**：能行动，但缺少显式的中间推理。
- **ReAct**：把推理和行动结合起来，更适合多步决策任务。

## 小结
ReAct 的重点不在于多写几句 thought，而在于让 thought 真正服务于 action，并让 observation 反过来修正后续推理。它适合需要“边查边想、边做边改”的任务。
